# Portfolio-Texte

## Ablauf

1. **Recherche:** URL via WebFetch analysieren (Branche, Zielgruppe, Technik, Umfang)
2. **Rückfragen:** Was unklar → nachfragen (Ausgangszustand, Herausforderung, messbare Ergebnisse)
3. **Schreiben:** 4-Teile-Struktur, 150-300 Wörter gesamt
4. **Review:** Text zeigen, Feedback einarbeiten

## Struktur

| Teil | Inhalt | Länge |
|---|---|---|
| **A: Einleitung** | Wer ist der Kunde, was wurde umgesetzt | 2-3 Sätze |
| **B: Ausgangslage** | Problem/Herausforderung vorher | 2-4 Sätze |
| **C: Lösung** | Technologien, Besonderheiten, USPs | 3-5 Sätze |
| **D: Ergebnis** | Messbare Resultate, Mehrwert | 2-3 Sätze |

## Stilguide

- SIE-Form, professionell aber nicht steif
- Konkrete Zahlen: "Ladezeit von 6s auf 1.8s" statt "deutlich schneller"
- Technologien nennen: "WordPress + Bricks Builder" statt "modernes CMS"
- **Verboten:** "einzigartig", "ganzheitlich", "maßgeschneidert", "revolutionär", Superlative ohne Beleg
- NUR bestätigte Fakten – nichts erfinden

## Output

Speichern als `portfolio-texte/{kundenname-slug}.md`
